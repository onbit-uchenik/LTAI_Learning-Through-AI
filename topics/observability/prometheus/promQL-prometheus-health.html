PromQL Queries for Prometheus Health

Memory & Disk

Prometheus container current memory usage. This gives the ballpark idea of how much data is stored as head series.

container_memory_working_set_bytes{pod=~"prometheus-prometheus-prometheus-0", container="prometheus"}

How much disk Prometheus is using (bytes)

prometheus_tsdb_storage_blocks_bytes{container="prometheus"}

OOM risk percentage. How close the container is to its memory limit.

container_memory_working_set_bytes{container="prometheus"} / container_spec_memory_limit_bytes{container="prometheus"}

PVC disk usage percentage for Prometheus persistent volume.

kubelet_volume_stats_used_bytes{persistentvolumeclaim=~"prometheus.*"} / kubelet_volume_stats_capacity_bytes{persistentvolumeclaim=~"prometheus.*"}

Head Block & Series

Total number of series in the Head Block (count)

prometheus_tsdb_head_series

20 most expensive metrics. Most expensive is defined as total number of unique series per metric.

topk(20,
  count by (__name__) ({__name__=~".+"})
)

WAL (Write-Ahead Log)

WAL disk size. A bloated WAL means checkpoint failures or high churn, and causes slow restarts.

prometheus_tsdb_wal_storage_size_bytes

WAL checkpoint creation failures.

prometheus_tsdb_checkpoint_creations_failed_total

WAL truncation failures.

prometheus_tsdb_wal_truncations_failed_total

Compaction

Compaction failures. Failed compactions mean blocks aren't being merged, leading to query slowness and disk bloat.

prometheus_tsdb_compactions_failed_total

Ingestion

Samples ingested per second. Best metric for understanding ingestion load.

rate(prometheus_tsdb_samples_appended_total[5m])

Out of order samples rejected per second. High rate means wasted scrapes or misconfigured targets.

rate(prometheus_tsdb_out_of_order_samples_total[5m])

Series Churn

Series churn - creation rate. High creation rate means high cardinality churn and memory pressure.

rate(prometheus_tsdb_head_series_created_total[5m])

Series churn - removal rate. Pair with creation rate to understand net series growth.

rate(prometheus_tsdb_head_series_removed_total[5m])

Scrape Health

Scrape targets that are down.

up == 0

Scrape duration per target. Slow scrapes can cause timeouts and data gaps.

scrape_duration_seconds

Per-target series count. Helps find targets pushing too many series.

scrape_samples_scraped

Query Performance

Query performance. Slow queries consume memory and CPU.

prometheus_engine_query_duration_seconds

Config

Config reload status. 0 means the last reload failed and a broken config is being silently ignored.

prometheus_config_last_reload_successful
